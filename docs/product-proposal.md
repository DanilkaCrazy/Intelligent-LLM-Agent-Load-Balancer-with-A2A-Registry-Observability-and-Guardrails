# Product Proposal: Agent Platform

## Обоснование идеи
В современных AI-системах агенты (A2A) часто общаются друг с другом и с различными LLM-провайдерами. Отсутствие стандартизированного слоя маршрутизации приводит к дублированию кода, сложностям с мониторингом затрат и уязвимостям безопасности. Платформа решает задачу создания единого шлюза (AI Gateway), который берет на себя балансировку, безопасность и телеметрию.

## Цель и метрики
**Цель:** Создать надежный, масштабируемый и безопасный шлюз для A2A-взаимодействия и доступа к LLM.

**Метрики успеха:**
- *Продуктовые:* Поддержка 3+ LLM-провайдеров "из коробки", удобный дашборд для управления.
- *Агентские:* Успешная регистрация и discovery агентов через API.
- *Технические:* p95 latency добавленная платформой < 50ms, 100% покрытие запросов метриками (TTFT, TPOT, Cost), успешная блокировка 99% базовых prompt-injections.

## Потенциальные сценарии использования
1. **Маршрутизация по стоимости:** Агент запрашивает модель `gpt-3.5-turbo`. Платформа видит, что локальная модель `llama-3-8b` (которая совместима по API) свободна и стоит дешевле, и прозрачно маршрутизирует запрос туда.
2. **Failover:** Провайдер Anthropic недоступен (503). Платформа автоматически переключает запрос на OpenAI с эквивалентной моделью.
3. **Edge-кейс (DDoS или спам):** Злонамеренный агент отправляет тысячи запросов. Платформа отсекает их по Rate Limit и Guardrails.

## Ограничения
- **Технические:** Добавленная задержка (overhead) не должна превышать 50ms. Платформа должна выдерживать до 1000 RPS на один инстанс.
- **Операционные:** Бюджет на инстансы платформы минимален (развертывание в Docker/K8s).

## Архитектурный набросок
- **API Gateway / Router:** Принимает запросы, проверяет авторизацию.
- **Guardrails Module:** Анализирует промпты до отправки к LLM.
- **Balancer:** Выбирает оптимального провайдера на основе Health и Latency.
- **Telemetry Service:** Асинхронно собирает метрики и отправляет в Prometheus.
- **Registry:** In-memory (в PoC) или Redis/Postgres хранилище для агентов и провайдеров.

## Data Flow
1. Агент -> Платформа (Auth, Guardrails).
2. Платформа -> Выбор провайдера (Balancer).
3. Платформа -> LLM Provider (OpenAI/Anthropic).
4. LLM Provider -> Платформа (Сбор метрик: токены, время).
5. Платформа -> Агент.
*Делегировано LLM:* Генерация текста. *Не делегировано:* Маршрутизация, безопасность, подсчет стоимости.
